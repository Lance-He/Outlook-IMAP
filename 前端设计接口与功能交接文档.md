# 前端功能与接口交接文档

## 1. 文档目的

这份文档只描述两件事：

1. 当前系统需要前端承接的功能
2. 当前后端已经提供的真实接口

不包含：

- 界面布局要求
- 样式要求
- 视觉方向
- 页面组件设计建议


## 2. 系统定位

这是一个 Outlook IMAP OAuth 邮件接收工具。

当前系统的核心工作方式：

1. 批量导入多个邮箱账户
2. 后端逐条验证邮箱凭据
3. 后端统一调度账户同步
4. 前端不主动轮询
5. 后端通过 `SSE` 将新邮件和状态变化推送到前端
6. 前端负责展示账户状态、邮件列表、邮件详情、原始 MIME 内容


## 3. 当前前端必须覆盖的功能

### 3.1 初始化

页面初始化时需要完成：

1. 获取系统设置
2. 获取账户列表
3. 如果账户列表非空，默认选中一个账户
4. 加载该账户下的邮件列表


### 3.2 账户导入

前端需要支持批量导入。

导入输入格式固定为：

```text
邮箱----密码----client_id----refresh_token
```

要求：

1. 支持按换行批量输入
2. 支持调用后端接口获取逐条结果
3. 展示每一行的成功 / 失败状态
4. 展示失败原因

注意：

- 当前真实后端会对每条账户做在线验证
- 不是只做文本格式校验


### 3.3 账户列表展示

前端需要展示每个账户的以下信息：

- `id`
- `email`
- `displayName`
- `note`
- `verifyStatus`
- `runtimeStatus`
- `pullIntervalSec`
- `enabled`
- `lastSyncAt`
- `lastError`
- `totalMessages`
- `lastMessageAt`


### 3.4 账户启停

前端需要支持：

1. 启动某个账户
2. 停止某个账户

说明：

- 启动后由后端调度器负责同步
- 前端不负责发起持续同步循环


### 3.5 修改拉取间隔

前端需要支持修改单个账户的拉取间隔。

当前系统默认支持的秒数值为：

- `30`
- `60`
- `120`
- `300`
- `600`
- `900`


### 3.6 邮件列表

前端需要支持加载并展示指定账户下的邮件列表。

每封邮件当前需要展示的字段：

- `id`
- `accountId`
- `subject`
- `fromName`
- `fromAddress`
- `receivedAt`
- `textPreview`
- `hasHtml`
- `hasAttachments`

前端当前还需要支持关键词搜索，搜索范围包括：

- 邮件主题
- 发件人名称
- 发件人地址
- 文本预览


### 3.7 邮件详情

前端需要支持查看单封邮件的详情。

当前详情数据包含：

- 邮件基础摘要
- `headers`
- `htmlContent`
- `textContent`

同时还需要支持查看原始邮件内容：

- 原始 MIME / `.eml`


### 3.8 实时更新

前端必须接入实时事件流。

要求：

1. 页面建立 `SSE` 连接
2. 收到事件后更新账户列表
3. 如果事件属于当前账户，更新邮件列表
4. 如果事件包含新邮件 `messageId`，允许自动选中新邮件

注意：

- 当前系统不要求前端主动轮询
- 后端心跳会定时发送，前端需要容忍断线重连


### 3.9 错误提示

前端需要对以下错误进行提示：

- 初始化失败
- 导入失败
- 邮件列表加载失败
- 邮件详情加载失败
- 原始 MIME 加载失败
- SSE 断线或异常
- 账户启停失败
- 间隔更新失败


## 4. 当前真实数据结构

### 4.1 Account

```ts
type Account = {
  id: string;
  email: string;
  displayName: string;
  note: string;
  verifyStatus: string;
  runtimeStatus: string;
  pullIntervalSec: number;
  enabled: boolean;
  lastSyncAt: string | null;
  lastError: string | null;
  totalMessages: number;
  lastMessageAt: string | null;
};
```


### 4.2 MessageSummary

```ts
type MessageSummary = {
  id: string;
  accountId: string;
  subject: string;
  fromName: string | null;
  fromAddress: string | null;
  receivedAt: string | null;
  textPreview: string | null;
  hasHtml: boolean;
  hasAttachments: boolean;
};
```


### 4.3 MessageDetail

```ts
type MessageDetail = MessageSummary & {
  headers: Record<string, string>;
  htmlContent: string | null;
  textContent: string | null;
};
```


### 4.4 ImportResult

```ts
type ImportResult = {
  lineNumber: number;
  email: string | null;
  status: "success" | "error";
  message: string;
  accountId?: string;
};
```


### 4.5 Settings

```ts
type Settings = {
  defaultPullIntervalSec: number;
  maxConcurrency: number;
  retryBackoffSec: number;
  pushMode: string;
};
```


### 4.6 SSE 事件

```ts
type AppEvent = {
  type: string;
  accountId?: string;
  messageId?: string;
};
```

说明：

- 当前后端事件中还可能带 `status`、`insertedCount`
- 前端当前主依赖字段是 `type`、`accountId`、`messageId`


## 5. 当前后端真实接口

以下接口是当前已经在后端实现的真实接口。

---

### 5.1 健康检查

#### `GET /api/health`

返回示例：

```json
{
  "status": "ok",
  "pushMode": "SSE"
}
```

---

### 5.2 获取系统设置

#### `GET /api/settings`

返回示例：

```json
{
  "defaultPullIntervalSec": 60,
  "maxConcurrency": 3,
  "retryBackoffSec": 30,
  "pushMode": "SSE"
}
```

---

### 5.3 获取账户列表

#### `GET /api/accounts`

返回值：

- `Account[]`

---

### 5.4 导入预校验

#### `POST /api/accounts/import/preview`

请求体：

```json
{
  "bulkText": "..."
}
```

返回值：

- 逐条校验结果数组

说明：

- 后端已实现
- 当前旧前端未使用
- 新前端如需两步式导入，可直接使用

---

### 5.5 导入并提交

#### `POST /api/accounts/import/commit`

请求体：

```json
{
  "bulkText": "a@outlook.com----pass----client_id----refresh_token"
}
```

返回示例：

```json
{
  "results": [
    {
      "lineNumber": 1,
      "email": "a@outlook.com",
      "status": "success",
      "message": "验证通过，已写入账户列表",
      "accountId": "xxx"
    },
    {
      "lineNumber": 2,
      "email": "b@outlook.com",
      "status": "error",
      "message": "该邮箱已存在"
    }
  ]
}
```

后端当前会执行：

1. 文本格式校验
2. 邮箱是否已存在校验
3. Outlook OAuth 凭据验证
4. IMAP 收件箱访问验证

当前验证并发数固定为：

- `3`

---

### 5.6 修改账户

#### `PATCH /api/accounts/:id`

请求体可包含：

```json
{
  "displayName": "新名称",
  "note": "备注",
  "pullIntervalSec": 60
}
```

返回示例：

```json
{
  "success": true
}
```

当前前端实际使用的字段只有：

- `pullIntervalSec`

---

### 5.7 启动账户

#### `POST /api/accounts/:id/start`

返回示例：

```json
{
  "success": true
}
```

说明：

- 后端会立刻进入调度流程

---

### 5.8 停止账户

#### `POST /api/accounts/:id/stop`

返回示例：

```json
{
  "success": true
}
```

---

### 5.9 获取指定账户邮件列表

#### `GET /api/accounts/:id/messages`

返回值：

- `MessageSummary[]`

---

### 5.10 获取邮件详情

#### `GET /api/messages/:id`

返回值：

- `MessageDetail`

返回示例结构：

```json
{
  "id": "msg-id",
  "accountId": "account-id",
  "subject": "邮件主题",
  "fromName": "发件人",
  "fromAddress": "sender@example.com",
  "receivedAt": "2026-04-09T12:00:00.000Z",
  "textPreview": "摘要",
  "hasHtml": true,
  "hasAttachments": false,
  "headers": {
    "subject": "邮件主题"
  },
  "htmlContent": "<html>...</html>",
  "textContent": "正文"
}
```

---

### 5.11 获取邮件原始 MIME

#### `GET /api/messages/:id/raw`

返回值：

- 原始 `.eml` / MIME 文本

响应类型：

- `text/plain`

---

### 5.12 实时事件流

#### `GET /api/events/stream`

协议：

- `SSE`

响应头关键点：

- `Content-Type: text/event-stream`
- `Cache-Control: no-cache, no-transform`

说明：

- 后端每 25 秒发一次心跳
- 前端应通过 `EventSource` 连接


## 6. 当前实时事件类型

当前后端会推送以下事件：

- `connected`
- `account_verified`
- `account_status_changed`
- `sync_started`
- `sync_succeeded`
- `sync_failed`
- `new_message`

当前事件可能附带字段：

- `type`
- `accountId`
- `messageId`
- `status`
- `insertedCount`


## 7. 当前真实交互流程

### 7.1 页面启动流程

1. `GET /api/accounts`
2. `GET /api/settings`
3. 如果有账户，自动选中一个账户
4. `GET /api/accounts/:id/messages`
5. 建立 `SSE` 连接


### 7.2 切换账户流程

1. 用户选择账户
2. 请求该账户邮件列表
3. 默认选中列表第一封邮件
4. 请求邮件详情


### 7.3 查看邮件详情流程

1. 用户选择邮件
2. `GET /api/messages/:id`
3. 默认展示详情数据
4. 若用户进一步需要原始内容，再请求 `/raw`


### 7.4 导入流程

1. 用户粘贴多行文本
2. 前端发送 `bulkText`
3. 后端逐条验证
4. 前端展示逐条结果
5. 成功账户出现在账户列表


### 7.5 实时更新流程

1. 前端建立 `SSE`
2. 后端调度器同步账户
3. 同步状态变化或新邮件到达时推送事件
4. 前端收到事件后：
   - 刷新账户列表
   - 如命中当前账户，则刷新邮件列表
   - 如有新邮件且当前未选中邮件，可自动选中


## 8. 当前系统已实现与未实现边界

### 8.1 当前已实现

- 获取设置
- 获取账户列表
- 批量导入并验证
- 账户启停
- 修改拉取间隔
- 获取邮件列表
- 获取邮件详情
- 获取原始 MIME
- SSE 实时推送


### 8.2 当前未实现或前端未接入

- 删除账户接口
- 账户备注编辑 UI
- 显示名编辑 UI
- 两步式 preview + commit 导入流程
- 高级筛选
- 分页
- 排序切换
- 手动立即同步按钮


## 9. 交接结论

给新的前端设计/实现同学，只需要按下面这条原则接：

1. 保持现有接口不变
2. 保持 `SSE` 实时推送机制不变
3. 保持导入格式不变
4. 完整覆盖账户、邮件列表、邮件详情、原始 MIME、错误提示这些功能

除了这些功能与接口契约，其余内容都不在本文件约束范围内。
